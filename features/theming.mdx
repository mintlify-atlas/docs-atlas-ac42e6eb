Bodequita supports **light** and **dark** themes. The active theme is persisted across app restarts and can be toggled from the Profile screen.

## ThemeManager

`ThemeManager` is the single source of truth for the current theme preference. It is instantiated in `MainActivity` and passed down through `MainNavigation` to the `ProfileScreen`.

```kotlin
class ThemeManager(context: Context) {
    private val prefs = context.getSharedPreferences("mini_tienda_theme", Context.MODE_PRIVATE)
    private val THEME_KEY = "is_dark_theme"

    private val _isDarkTheme = MutableStateFlow(getSavedTheme())
    val isDarkTheme: StateFlow<Boolean> = _isDarkTheme.asStateFlow()

    private fun getSavedTheme(): Boolean {
        return prefs.getBoolean(THEME_KEY, false) // false = light theme by default
    }

    fun toggleTheme() {
        val newTheme = !_isDarkTheme.value
        _isDarkTheme.value = newTheme
        prefs.edit().putString(THEME_KEY, newTheme.toString()).apply()
    }

    fun setTheme(isDark: Boolean) {
        _isDarkTheme.value = isDark
        prefs.edit().putBoolean(THEME_KEY, isDark).apply()
    }
}
```

**Persistence mechanism:** `SharedPreferences` file `mini_tienda_theme`, key `is_dark_theme` (Boolean). The default value is `false` (light theme).

The `isDarkTheme` `StateFlow` is collected in `MainActivity` and passed as the `darkTheme` parameter to `MiniTiendaTheme`.

<Note>
  `toggleTheme()` uses `putString` while `setTheme()` uses `putBoolean`. Both methods update the in-memory `StateFlow` immediately so the UI reacts without requiring an app restart.
</Note>

## Theme composable

`MiniTiendaTheme` wraps `MaterialTheme` and selects the correct color scheme based on the `darkTheme` parameter:

```kotlin
@Composable
fun MiniTiendaTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    content: @Composable () -> Unit
) {
    val colorScheme = if (darkTheme) DarkColorScheme else LightColorScheme
    val view = LocalView.current

    if (!view.isInEditMode) {
        SideEffect {
            val window = (view.context as Activity).window
            window.statusBarColor = colorScheme.primary.toArgb()
            WindowCompat.getInsetsController(window, view).isAppearanceLightStatusBars = !darkTheme
        }
    }

    MaterialTheme(
        colorScheme = colorScheme,
        typography = Typography,
        content = content
    )
}
```

The status bar color and icon appearance are updated via `SideEffect` whenever the theme changes.

| `displayMedium` | Medium | 28 sp | 42 sp | Section titles |
| `displaySmall` | Medium | 24 sp | 36 sp | Card titles |
| `headlineMedium` | Medium | 16 sp | 24 sp | Subsection titles |
| `bodyLarge` | Normal | 16 sp | 24 sp | Body text |
| `bodySmall` | Normal | 14 sp | 20 sp | Metadata and descriptions |
| `labelLarge` | Medium | 16 sp | 24 sp | Labels |
| `labelMedium` | Medium | 16 sp | 24 sp | Button text |

## Switching the theme from the Profile screen

The `ProfileScreen` receives the `ThemeManager` instance and collects `isDarkTheme` as state:

```kotlin
ProfileScreen(
    navController,
    onMenuClick = openDrawer,
    themeManager = themeManager
)
```

Inside the screen, collecting the flow and calling `themeManager.toggleTheme()` from a switch or button is sufficient to persist and apply the change immediately across all recompositions that observe `MiniTiendaTheme`.

<Tip>
  Because `ThemeManager.isDarkTheme` is a `StateFlow`, any composable that calls `themeManager.isDarkTheme.collectAsState()` will automatically recompose when the theme changes — no manual invalidation is needed.
</Tip>
