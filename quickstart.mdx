## Prerequisites

Before you begin, make sure the following tools are installed:

- **Android Studio** Hedgehog (2023.1.1) or later
- **JDK 17** — the project sets `sourceCompatibility = JavaVersion.VERSION_17` and `jvmTarget = "17"` in `app/build.gradle.kts`
- **Android SDK** with API level 25 (Android 7.1 Nougat) or higher — `minSdk = 25`, `targetSdk = 36`
- **Git**

<Steps>
  <Step title="Clone the repository">
    Open a terminal and clone the project:

    ```bash
    git clone https://github.com/your-org/bodequita.git
    cd bodequita
    ```
  </Step>

  <Step title="Open the project in Android Studio">
    1. Launch Android Studio.
    2. Select **File → Open** and navigate to the cloned directory.
    3. Click **OK** and wait for the initial Gradle sync to complete.

    <Note>
      Android Studio will automatically download all declared dependencies, including Room 2.6.1 and Vico 1.13.0, during the first sync. Make sure you have an active internet connection.
    </Note>
  </Step>

  <Step title="Configure an emulator or physical device">
    The app targets API 25+. You can use either:

    - **Emulator**: In Android Studio, go to **Device Manager → Create Device** and choose a Pixel profile with an API 25+ system image.
    - **Physical device**: Enable **USB Debugging** on your Android device (Settings → Developer Options) and connect via USB.
  </Step>

  <Step title="Build the project">
    From the Android Studio toolbar, select **Build → Make Project** (or press `Ctrl+F9` / `Cmd+F9`). You can also build from the command line:

    ```bash
    ./gradlew assembleDebug
    ```

    The debug APK is output to `app/build/outputs/apk/debug/app-debug.apk`.
  </Step>

  <Step title="Run the app">
    Press **Run** (the green play button) or use:

    ```bash
    ./gradlew installDebug
    ```

    The app launches on your selected device or emulator.
  </Step>

  <Step title="First launch walkthrough">
    Follow this path through the app on first launch:

    1. **Login screen** — The app starts at the `login` route. Enter any credentials to proceed to the main app (authentication is local).
    2. **Dashboard** — You land on the dashboard (`dashboard` route), which shows today's sales totals, units sold, and the weekly sales chart. The chart will be empty until you record sales.
    3. **Create your first product** — Open the navigation drawer (hamburger icon) and tap **Productos**. Tap the add button to navigate to `product_registration`. Fill in the product name, code, price, stock quantity, and optionally assign a category. Tap **Save**.
    4. **Make your first sale** — From the drawer, tap **Ventas**. Enter the product code and quantity, then tap **Add to cart**. Review the cart total and tap **Finalize sale**. The sale is persisted in a Room transaction that also decrements the product's stock.
  </Step>
</Steps>

<Tip>
  The Room database file is stored on-device at:
  `data/data/com.trabajo.minitienda/databases/minitienda.db`

  You can inspect it during development using **Android Studio → App Inspection → Database Inspector** while the app is running on a connected device or emulator.
</Tip>
