title: Supplier
description: Room entity and DAO interface for suppliers (proveedores).
---

The `Supplier` entity represents a vendor that supplies products. It is stored in the `proveedor` table.

## Entity definition

```kotlin
@Entity(
    tableName = "proveedor",
    indices = [
        Index(value = ["ruc"], unique = true),
        Index(value = ["nombre"], unique = true)
    ]
)
data class Supplier(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    val ruc: String,
    val nombre: String,
    val telefono: String? = null
)
```

## Fields

<ParamField path="id" type="Int" required>
  Auto-generated primary key.
</ParamField>

<ParamField path="ruc" type="String" required>
  Supplier tax ID (RUC). Must be unique across all suppliers.
</ParamField>

<ParamField path="nombre" type="String" required>
  Supplier business name. Must be unique across all suppliers.
</ParamField>

<ParamField path="telefono" type="String?" default="null">
  Optional contact phone number.
</ParamField>

## Indexes

| Column | Unique |
|---|---|
| `ruc` | Yes |
| `nombre` | Yes |

<Note>
  Both `ruc` and `nombre` have unique constraints. Attempting to insert a supplier with a duplicate `ruc` via `insertIgnore` will silently discard the row and return `-1L`.
</Note>

## SupplierDao

```kotlin
@Dao
interface SupplierDao {

    @Query("SELECT * FROM proveedor ORDER BY nombre ASC")
    fun observeAll(): Flow<List<Supplier>>

    @Insert(onConflict = OnConflictStrategy.IGNORE)
    suspend fun insertIgnore(s: Supplier): Long

    @Query("UPDATE proveedor SET nombre = :nuevo WHERE id = :id")
    suspend fun rename(id: Int, nuevo: String): Int

    @Query("SELECT * FROM proveedor WHERE ruc = :ruc LIMIT 1")
    suspend fun findByRuc(ruc: String): Supplier?

    @Query("UPDATE proveedor SET nombre = :nombre, telefono = :telefono WHERE ruc = :ruc")
    suspend fun updateByRuc(ruc: String, nombre: String, telefono: String?): Int

    @Delete
    suspend fun delete(s: Supplier)
}
```

### Method reference

| Method | Return type | Description |
|---|---|---|
| `observeAll()` | `Flow<List<Supplier>>` | Reactive stream of all suppliers sorted by name ascending. |
| `insertIgnore(s)` | `Long` | Inserts the supplier, ignoring conflicts on `ruc` or `nombre`. Returns `-1L` on conflict. |
| `rename(id, nuevo)` | `Int` | Updates the `nombre` of the supplier with the given `id`. Returns rows affected. |
| `findByRuc(ruc)` | `Supplier?` | Looks up a supplier by `ruc`. Returns `null` if not found. |
| `updateByRuc(ruc, nombre, telefono)` | `Int` | Updates `nombre` and `telefono` for the supplier matching `ruc`. Returns rows affected. |
| `delete(s)` | `Unit` | Deletes the supplier row matching the entity's primary key. |

<Tip>
  In `PurchasesViewModel.upsertSupplierByRuc`, a supplier is created by calling `insertIgnore`. If it returns `-1L` the supplier already exists and the operation is a no-op. For full upsert behavior (update if exists), call `updateByRuc` after checking the return value.
</Tip>
